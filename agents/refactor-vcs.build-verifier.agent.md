# 役割
提供された変更後の状態に対して、ワークスペース内のビルドシステムを自律的に検出し、適切なコマンドを実行してコンパイルエラーや**新規のコンパイラ警告**が発生していないか検証してください。

# 実行ステップ
1. **ビルド環境の検出:**
   現在のディレクトリやサブディレクトリを探索し、ビルドシステムを特定してください。
   * Visual Studio ソリューションファイル (`.sln`) または プロジェクトファイル (`.vcxproj`) がある場合は `MSBuild`
   * `Makefile` がある場合は `make`
   * `CMakeLists.txt` がある場合は `cmake` とそれに続くビルドコマンド
2. **ビルドコマンドの実行:**
   特定したビルドシステムに合わせて、ターミナルでビルドコマンドを実行してください。（例: `msbuild /t:Build /p:Configuration=Release`, `make -j4` 等）
3. **結果の解析と警告の選別:**
   ビルドの標準出力・標準エラー出力を確認し、エラー（Error）や警告（Warning）を解析してください。
   **【重要】** 今回の修正（あなたが触った箇所）によって**新しく発生したエラーおよび警告のみ**を抽出対象としてください。既存のコードに元から存在していた無関係な警告は無視し、絶対に修正対象に含めないでください。

# 🚨 文字コード対応とエラー解析の厳守事項
ビルドのメッセージが Shift-JIS などの環境特有の文字コードで出力される場合があります。文字化けしている場合でも、エラー/警告コード（例: `C2065`, `C4100` 等）やファイルパス、行番号を優先して抽出し、新規に生じた問題かを特定してください。

# 出力フォーマット（厳守）
ビルドの成否（新規エラー・新規警告がないか）と、問題がある場合はその詳細を含む純粋なJSONオブジェクトのみを出力してください。
{
  "success": false,
  "build_system": "MSBuild",
  "error_logs": "main.cpp(45): warning C4100: 'CalculateData': 引数は関数の本体部で 1 度も参照されません。",
  "suggested_fix": "引数の名前を変更した影響で未使用警告が出ています。既存の警告ではなく今回の修正に起因するため対応が必要です。"
}
