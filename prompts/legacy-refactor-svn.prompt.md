# 指示
提供する `svn diff` の結果（または現在添付している差分テキスト）をもとに、新規追加された変数および関数の命名規約チェックとコード修正を、エージェントモードで自律的に実行してください。
各工程は専門のサブエージェントに委譲し、エージェント間では「純粋なJSON配列」のみを受け渡ししてコンテキストを節約してください。

# 実行ステップ（ワークフロー）
1. **差分抽出:** `svn-diff-analyzer` に差分テキストを渡し、新規定義のみを抽出させてください。
2. **規約チェック:** ステップ1のJSON出力を `convention-checker` に渡し、`coding-guides.md` と照合させてください。
3. **影響・整合性調査:** ステップ2の出力を `impact-analyzer` に渡し、修正が必要な全ファイルパスと行番号を特定させてください。
4. **コード修正:** ステップ3の出力を `code-modifier` に渡し、ファイルを実際に修正させてください。
5. **再チェックと反復:** 修正完了後、あなた自身でコードの整合性を確認し、問題があれば整合性がとれるまで反復してください。

# 🚨 厳守事項
* 各サブエージェントが連携する際、`fileencoding` の値（Shift-JISなど）を絶対に見失わないでください。
* 修正対象は「新規に追加・定義された変数と関数」のみに限定（ボーイスカウトルール）してください。
