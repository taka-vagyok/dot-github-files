# Agent Spec

本ドキュメントは、GitHub Copilot 向けのエージェント (Agent) を作成する際のルールとベストプラクティスを定義したものです。
Agentは、複数のスキルやプロンプトを統合し、タスクの全体的な制御やオーケストレーションを担当します。

プロンプトやスキルの詳細な仕様については、以下のドキュメントを参照してください。
*   [Prompt Spec](./prompt-spec.md)
*   [Skill Spec](./skill-spec.md)

## 1. 役割の分割（オーケストレーターと専門エージェント）

*   **神エージェントの排除**:
    1つのエージェントに「要件定義」「コーディング」「テスト」などの複数の役割を持たせることは厳禁です。必ず役割ごとにエージェントを分割してください。
*   **オーケストレーターの責務 (Spec & Test First)**:
    進行管理を行うオーケストレーターは、専門エージェントに作業を振る前に、必ず「満たすべき仕様 (Spec)」と「完了条件・受入テスト (Test)」を定義しなければなりません。
*   **反復プロセス (Iteration) の明記**:
    専門エージェントから上がってきた成果物をそのまま返すのではなく、事前に定義したテスト基準と照らし合わせてレビューし、基準に達していなければフィードバックを与えて修正させる「反復ループ」をタスク手順に明記します。

## 2. エージェントの目標と制約

*   **目標の明確化**:
    専門エージェントが解決すべきタスクの最終的な目標を明確に定義してください。
*   **制約事項の明記**:
    エージェントの自律性を適切にコントロールするため、「やってはいけないこと」や「出力のトーン＆マナー」を明確に記述します。
*   **自律性と判断のガイドライン**:
    複数のステップで構成されるタスクにおいて、エージェントが自律的に判断を下すための指針を定めます。

## 3. スキルとプロンプトの活用

*   **利用可能なスキルの範囲・優先順位**:
    エージェントが使用可能なツール (Skill) を定義し、どの順序や条件でそれらを呼び出すべきかを設計します。
*   **説明 (Description) への発火条件の明記**:
    エージェント自体の `description` には、どのような状況やユーザーの意図でそのエージェントを呼び出すべきか、明確な「発火条件」を記述してください。これにより、適切なタイミングでエージェントが呼び出されるようになります。

## 4. フロー制御と堅牢性

*   **依存関係の整理**:
    タスク実行の各ステップ間の依存関係を整理し、一貫したワークフローを構築します。
*   **例外処理と回復手段**:
    スキルの実行失敗や予期せぬエラーに備え、代替手段や安全に終了するための回復フロー (Fallback mechanisms) を設計に組み込みます。
