# Agent Spec

本ドキュメントは、GitHub Copilot 向けのエージェント (Agent) を作成する際のルールとベストプラクティスを定義したものです。
Agentは、複数のスキルやプロンプトを統合し、タスクの全体的な制御やオーケストレーションを担当します。

プロンプトやスキルの詳細な仕様については、以下のドキュメントを参照してください。
*   [Prompt Spec](./prompt-spec.md)
*   [Skill Spec](./skill-spec.md)

## 1. 役割の分割（オーケストレーターと専門エージェント）

*   **神エージェントの排除**:
    1つのエージェントに「要件定義」「コーディング」「テスト」などの複数の役割を持たせることは厳禁です。必ず役割ごとにエージェントを分割してください。
*   **オーケストレーターの責務 (Spec & Test First)**:
    進行管理を行うオーケストレーターは、専門エージェントに作業を振る前に、必ず「満たすべき仕様 (Spec)」と「完了条件・受入テスト (Test)」を定義しなければなりません。
*   **反復プロセス (Iteration) の明記**:
    専門エージェントから上がってきた成果物をそのまま返すのではなく、事前に定義したテスト基準と照らし合わせてレビューし、基準に達していなければフィードバックを与えて修正させる「反復ループ」をタスク手順に明記します。

## 2. エージェントの目標と制約

*   **目標の明確化**:
    専門エージェントが解決すべきタスクの最終的な目標を明確に定義してください。
*   **制約事項の明記**:
    エージェントの自律性を適切にコントロールするため、「やってはいけないこと」や「出力のトーン＆マナー」を明確に記述します。

## 3. スキルとプロンプトの活用

*   **利用可能なスキルの範囲・優先順位**:
    エージェントが使用可能なツール (Skill) を定義し、どの順序や条件でそれらを呼び出すべきかを設計します。
*   **説明 (Description) への発火条件の明記**:
    エージェント自体の `description` には、どのような状況やユーザーの意図でそのエージェントを呼び出すべきか、明確な「発火条件」を記述してください。これにより、適切なタイミングでエージェントが呼び出されるようになります。

## 4. フロー制御と堅牢性

*   **依存関係の整理**:
    タスク実行の各ステップ間の依存関係を整理し、一貫したワークフローを構築します。
*   **例外処理と回復手段**:
    スキルの実行失敗や予期せぬエラーに備え、代替手段や安全に終了するための回復フロー (Fallback mechanisms) を設計に組み込みます。

## 5. レビューチェックリスト (Review Checklist)

AIコンポーネントをレビューするエージェント（例: `ai-component-manager.reviewer.agent`）は、エージェントファイルが以下の基準を満たしているか確認してください。

*   [ ] **拡張子と配置**: `agents/` ディレクトリ配下にあり、拡張子が `.agent.md` であるか。
*   [ ] **フロントマター**: YAML形式で `name`, `description`, `tools` が定義されているか。不要なツールが付与されていないか。
*   [ ] **責務の分離 (Orchestrator)**: オーケストレーターである場合、直接ファイル編集（`edit` ツールなど）を行っていないか。タスクを専門エージェントに委譲（`agent/runSubagent`）し、結果をレビューする「反復ループ（Iteration）」が手順に組み込まれているか。
*   [ ] **責務の分離 (Specialist)**: 専門エージェントである場合、特定のドメイン知識に基づく「推論」や「判断」に集中しており、全体の進行管理や無関係な外部操作（神エージェント）を行っていないか。
*   [ ] **テスト駆動アプローチ**: 実装を開始する前に、必ず「仕様（Spec）とテスト条件（Acceptance Criteria）」を定義・確認するよう指示されているか。

## 6. エージェントのサンプルフォーマット

オーケストレーターエージェントの基本フォーマットは以下の通りです。Markdownのフロントマター形式（拡張子: `.agent.md`）を使用し、`tools`に必要最低限のツールのみを定義します。

```markdown
---
name: "my-task.orchestrator.agent"
description: "〇〇タスクの進行を管理し、専門エージェントを指揮して課題を解決します。ユーザーからの入力が整理された状態で渡された際に呼び出されます。"
tools:
  - "agent/runSubagent"
  - "github/createIssue"
---

あなたは〇〇タスクのオーケストレーターです。自身で直接コードを書いたり編集したりせず（`edit`ツールの使用禁止）、以下の手順で専門エージェントに作業を委譲してください。

1. 【Spec & Test First】
   ユーザーの要件から「満たすべき仕様」と「完了条件（受入テスト）」を明確に定義する。
2. 【実行委譲】
   定義した仕様とテストを `agent/runSubagent` ツールを使用して `my-task.specialist` に渡し、作業を実行させる。
3. 【レビューと反復】
   専門エージェントから上がってきた成果物を、1で定義した「完了条件」と照らし合わせてレビューする。基準に達していない場合はフィードバックを与え、再度修正させること（反復ループ）。
   基準を満たした場合のみ、ユーザーへ最終結果を報告する。
```
