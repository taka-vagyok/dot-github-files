# Agent Spec

本ドキュメントは、GitHub Copilot 向けのプロンプト、エージェント、およびスキルを作成する際のルールとベストプラクティスを定義したものです。

## 1. 全般的なルールとベストプラクティス

* **説明 (Description) への発火条件の明記**:
  プロンプト、エージェント、スキルの `description` (説明) には、どのような状況やユーザーの意図でそれを呼び出すべきか、明確な「発火条件」を記述してください。これにより、AIが適切なタイミングで呼び出すことが可能になります。
* **明確で簡潔な指示**:
  LLM が誤解しないよう、指示は明確かつ具体的に記述してください。期待する出力フォーマットがある場合は、例示とともに明記します。
* **依存関係の最小化**:
  可能な限り独立して動作するように設計し、外部や他の機能への依存は最小限にとどめてください。

## 2. 要素別の作成ルール

### プロンプト (Prompt)
* システムプロンプトにおいて、どのような役割を演じるか (Persona) と、タスク実行時の制約条件を明確に定義します。
* 期待する回答の精度を上げるため、必要に応じて参考となる例 (Few-shot prompting) を含めてください。

### エージェント (Agent)
* 複数のステップやスキルを組み合わせて複雑なタスクを遂行するため、タスクの全体的な目標と、使用可能なツールの範囲・優先順位を定義します。
* エージェントが自律的に判断を下すためのガイドライン（例：どのような場合にユーザーへ確認を求めるか）を含めてください。

### スキル (Skill)
* 単一の明確な機能（API呼び出しやスクリプト実行など）を提供するように設計します。
* 引数 (Parameters) を取る場合は、引数の型、必須かどうかのフラグ、および各引数の説明（何を渡すべきか）を正確に定義してください。
