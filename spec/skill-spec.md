# Skill Spec

本ドキュメントは、GitHub Copilot 向けのエージェントが実行するスキル (Skill/Tool) を作成する際のルールとベストプラクティスを定義したものです。
スキルはエージェントの推論能力だけでは解決できない、外部情報の取得やシステムへの副作用（書き込み等）を伴う処理を行うための「手足」となるツールです。

## 1. 明確な目的と粒度

*   **単一責務の原則**:
    1つのスキルは1つの明確なタスクに特化させるべきです。
*   **適切な抽象化**:
    複雑すぎる機能は避け、エージェントが組み合わせやすい粒度で設計します。

## 2. 発火条件（Trigger）の明確化

*   **説明文 (Description) への記述**:
    スキルの `description` は、単なるツールの説明ではなく、AIが「今このツールを使うべきか」を判断するためのトリガーです。発火条件を具体的に記述してください。
    *   *悪い例*: 「ウェブを検索します」
    *   *良い例*: 「ユーザーの入力に〇〇に関する要件が含まれている場合や、最新の情報を確認する必要がある場合は、【必ず】このスキルを使用してください。」
*   **直感的な名称**:
    ツール名は動詞と目的語を含めた簡潔で明確なものを使用してください。

## 3. 入出力と手順の厳密な定義

*   **入出力の厳密な定義**:
    スキルが受け取る引数のスキーマ (Input) と、エージェントに返すデータ形式 (Output) を正確かつ厳密に定義してください。
*   **手順の具体化**:
    スキル内でどのような手順で処理を実行するか、また例外やエラーが発生した場合にどう振る舞うかを明確に定義することで、システムの堅牢性を高めます。

## 4. エラーハンドリングと戻り値

*   **明確なエラーメッセージ**:
    スキルが失敗した場合は、エージェントが自己修復やリトライを行えるよう、「なぜ失敗したか」「次にどうすればよいか」を具体的に示すエラーメッセージを返却してください。
*   **一貫性のある戻り値**:
    戻り値はプレーンテキスト、JSON文字列など、パースしやすく一貫したフォーマットを採用します。
