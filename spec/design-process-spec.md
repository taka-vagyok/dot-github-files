# Design Process Spec

本ドキュメントは、GitHub Copilot 向けに新しい機能、エージェント、スキル、プロンプトを設計・作成、または既存のものを再構成する際の「思考プロセス（設計手法）」と「責務の分離」に関する仕様を定義したものです。
AIコンポーネントを設計するアーキテクト（例: `ai-component-manager.architect.agent.md`）は、必ずこの仕様に従って要件を分析し、コンポーネントを定義してください。

## 1. 最重要原則: 神コンポーネントの禁止

*   **神エージェントの禁止**: 1つの `.agent.md` ファイル内に、ユーザーとの対話（入力の整理）、全体の進行管理、複雑な推論・判定、そして具体的なコマンド実行（ファイル操作など）をすべて詰め込んではいけません。
*   **神スキルの禁止**: 1つの `SKILL.md` やその配下のスクリプト内に、要件定義や複雑な推論ロジックを含めてはいけません。スキルはあくまで「外部システムを操作する単機能の手足」でなければなりません。

## 2. 設計プロセス (Design Thinking Process)

ユーザーから「〇〇の機能を作って」と依頼された場合、以下のプロセスに従って設計を行ってください。

### ステップ 1: 要件の分解 (Requirement Analysis)
ユーザーの要求を以下の4つの要素に分解します。
1.  **入力/トリガー (Input/Trigger)**: ユーザーはどのようにこのタスクを開始するか？どのような情報を最初に提供するか？
2.  **進行管理/レビュー (Orchestration/Review)**: このタスクを達成するために、どのような手順を踏むか？作業結果をどのように検証・レビューするか？
3.  **推論/判断 (Reasoning/Logic)**: 特定の知識に基づいてコードを解析したり、良し悪しを判定したりする「考える」部分はどこか？
4.  **物理的作業/副作用 (Execution/Side-effects)**: ファイルを読み書きする、コマンドを実行する、APIを叩くといった「物理的な作業」は何か？

### ステップ 2: コンポーネントへの割り当て (Component Mapping)
分解した要素を、リポジトリの規約（`spec/*.md`）に基づき、以下のコンポーネントに割り当てます。

*   **Prompt (`prompts/*.prompt.md`)**
    *   *役割*: 「入力/トリガー」を担当。ユーザーとの対話の入り口となり、入力情報を整理してオーケストレーターに渡す。
*   **Orchestrator Agent (`agents/*.orchestrator.agent.md`)**
    *   *役割*: 「進行管理/レビュー」を担当。サブエージェント（専門家）やスキルを呼び出し、結果を検証して反復ループ（イテレーション）を回す監督役。自身では直接ファイル編集などの作業を行わない。
*   **Specialist Agent (`agents/*.specialist.agent.md`)**
    *   *役割*: 「推論/判断」を担当。オーケストレーターから依頼を受け、特定のドメイン知識に基づいて考える専門家。
*   **Skill (`skills/skill-name/SKILL.md`)**
    *   *役割*: 「物理的作業/副作用」を担当。コマンド実行やファイル操作を行う手足（ディレクトリとスクリプトで構成される）。

### ステップ 3: 既存コンポーネントのレビューと再構成 (Refactoring)
既存のファイルを見直す場合、ステップ2の基準に照らし合わせて以下を確認・修正します。
*   エージェントが `edit` や直接のシェルコマンド実行ツールを持っていないか？ -> 持っている場合は、その処理を **Skill** に分離する。
*   スキルの中に複雑な条件分岐やAIへの指示（プロンプト）が含まれていないか？ -> 含まれている場合は、その推論部分を **Specialist Agent** に引き剥がす。
*   プロンプトファイルの中に作業手順や推論プロセスが書かれていないか？ -> 書かれている場合は、それを **Orchestrator Agent** に移し、プロンプトは単なる入り口にする。

## 3. 設計書の出力形式

アーキテクトエージェントは、設計結果として以下を出力してください。
*   **要件の分析結果**: ユーザーの依頼をどのように4要素に分解したか。
*   **作成・修正すべきコンポーネントのリスト**: （例: `prompts/xxx.prompt.md`, `agents/xxx.orchestrator.agent.md`, `skills/xxx/SKILL.md`）
*   **各コンポーネントの責務と関係性**: 各ファイルが何を担当し、どのように連携するか（依存関係）。
