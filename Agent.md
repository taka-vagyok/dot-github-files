# Agent.md

本リポジトリは GitHub Copilot 向けのサンプルとなる Prompt、Agent、および Skill を定義・提供しています。
各要素の作り分け方、および命名規則などのベストプラクティスについて以下に説明します。

## プロンプト、エージェント、スキルの作り分けについて

*   **Prompt (プロンプト)**:
    特定のタスクにおいて、AI がどのような役割（Persona）で、どのような制約条件に従って回答すべきかを指示する静的なテキストです。単一の応答で完結するタスクに適しています。
*   **Agent (エージェント)**:
    複数の Prompt や Skill を組み合わせ、自律的に判断しながら複雑なプロセスやマルチステップのタスクを解決するものです。タスク全体の目標と、それに至る手順の制御を担当します。
*   **Skill (スキル)**:
    Agent が特定の機能（例：API 呼び出し、外部ツールの実行、特定のファイル操作など）を実行するための具体的な関数やツールです。単一の明確な機能を持つように設計します。

## 命名規則のベストプラクティス

一連の処理や関連する作業をまとめる際、それらを表す接頭単語（プレフィックス）をつけることをルールとします。これにより、機能がどのタスクに属しているか明確になります。

*   **特定の作業に関連する場合**:
    その作業を表す単語を区切り文字（`.` または `-`）で区切って接頭辞とします。
    例：`setup-environment`, `deploy.frontend`
*   **特定の作業に依存しない共通機能の場合**:
    `common` などの汎用的な単語を接頭辞とします。
    例：`common-logger`, `common.utils`
