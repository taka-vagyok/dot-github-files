# AI Agent Instructions (Agent.md)

このファイルは、本リポジトリで作業するAIエージェント（Copilot, Cursor, Devin等）のためのガイドラインです。コードの生成、リファクタリング、テストの作成を行う際は、必ず以下のルールとコンテキストに従ってください。

## 1. プロジェクトの概要 (Project Context)
* **プロジェクト名**: dot-github-files
* **目的**: GitHub Copilot 向けのサンプルの Prompt（プロンプト）、Agent（エージェント）、および Skill（スキル）を定義し、AI 開発のベストプラクティスに基づいた効果的な実装例を提供すること。
* **主要なドメイン**: AI エージェントのオーケストレーションと、再利用可能なスキルの構築。

## 2. 技術スタック (Tech Stack)
本リポジトリで管理するプロンプトやスクリプトは、一般的に以下の環境やフォーマットを想定しています。
* **Prompt/Agent**: YAML 形式 (`.prompt`, `.agent` など)
* **Skill**: (実装言語に依存しますが、一般的なスクリプト言語や TypeScript/Python などを想定)

## 3. ディレクトリ構成とアーキテクチャ (Architecture)
コードを追加する際は、オーケストレーターパターン（Prompt、Agent、Skill の厳密な責務分離）に従い、以下のディレクトリに配置してください。
* `prompts/`: ユーザーとシステムの接点となるエントリーポイント（ロジックを持たない）。
* `skills/`: エージェントが手足として使う具体的なツールや関数。
* `spec/`: プロンプト、エージェント、スキルの詳細な作成ルール（発火条件、サンプルフォーマットなど）を定めた仕様書。

アーキテクチャと各コンポーネントの作り分け方の本質（なぜ分割し、オーケストレーターパターンを採用するのか）や、詳細な作成ルールについては、以下のドキュメントを必ず参照してください。
* **[Agent Spec](./spec/agent-spec.md)**
* **[Prompt Spec](./spec/prompt-spec.md)**
* **[Skill Spec](./spec/skill-spec.md)**

## 4. コーディング規約 (Coding Standards)
* **命名規則**: プロンプト、エージェント、スキルにおいて、一連の処理や関連する作業をまとめる際は、それらを表す接頭単語をつけることをルールとします。
  * 特定の作業に関連する場合: `setup-environment`, `deploy.frontend` （`.` または `-` で区切る）
  * 特定の作業に依存しない共通機能の場合: `common-logger`, `common.utils`

## 5. テストと品質保証 (Testing & QA) ★最重要
AIエージェントは、本リポジトリにおいて**テスト駆動開発（TDD）**の原則に従うこと。
1. 新機能（新しい Agent や Skill）の実装を要求された場合、**コードを書く前に必ずテストコード（Spec）を先に作成・提案**すること。
2. テストは「正常系」だけでなく「異常系（境界値、エラー時の挙動）」を必ず含めること。
3. オーケストレーターは、専門エージェントから上がってきた成果物を事前に定義したテスト基準と照らし合わせてレビューし、ダメなら再実行させる「反復ループ」を行うこと。

## 6. AIエージェントの振る舞い (Agent Behavior)
* 推測でコードを書かないこと。要件が不明確な場合や、依存関係がわからない場合は、実装を始める前に人間に質問すること。
* 破壊的変更（主要なエージェントの振る舞い変更、スキルのシグネチャ変更）を伴う場合は、必ず事前に警告を出すこと。
* 新しいプロンプト、エージェント、スキルを提案する際は、「なぜそのアプローチをとったのか（どの Spec に従ったのか）」の理由を簡潔に添えること。
